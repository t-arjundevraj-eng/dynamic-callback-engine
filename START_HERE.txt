═══════════════════════════════════════════════════════════════════════════════

✅ AUTOMATIC
   • Auto-discovery of vendors from vendor_callback_queue_config
   • Auto-creation of target _producer tables
   • Auto-scheduling of polling tasks
   • Auto-subscription to Kafka topics
   • Auto-recovery from failures

✅ HIGH-PERFORMANCE
   • 350 messages/second throughput (7 tasks × 50 rows)
   • 7 concurrent pollers (no blocking)
   • Async Kafka publishing
   • JDBC batch operations
   • Connection pooling

✅ PRODUCTION-GRADE
   • SQL injection prevention (validated identifiers + parameterized)
   • Exception handling (resilient, logged, no crashes)
   • Resource management (pooling, graceful shutdown)
   • Comprehensive logging (DEBUG, INFO, ERROR levels)
   • Zero hardcoded values (all from config)

✅ HIGHLY SCALABLE
   • Add vendors by DB config (no code changes)
   • Adjust polling frequency per vendor
   • Adjust batch size per vendor
   • Leverage 20-thread pool capacity
   • Monitor and tune easily

🔧 COMPONENTS
══════════════════════════════════════���════════════════════════════════════════

┌─ DynamicPollingManager ─────────────────────────────────────────────────────┐
│                                                                             │
│  @Service that initializes on startup                                      │
│  Creates ThreadPoolTaskScheduler (20 threads)                              │
│  Loads vendor_callback_queue_config (7 rows)                               │
│  Schedules VendorQueuePollerTask for each vendor                           │
│  Gracefully shuts down on app termination                                  │
│                                                                             │
│  When: Startup (@PostConstruct)                                            │
│  Output: 7 scheduled tasks ready to poll                                   │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ VendorQueuePollerTask ─────────────────────────────────────────────────────┐
│                                                                             │
│  Runnable task that executes every 1000ms                                  │
│  Polls source table: SELECT * LIMIT 50                                     │
│  Sends rows to Kafka topic                                                 │
│  Exception-safe (continues on failure)                                     │
│                                                                             │
│  When: Every 1000ms (forever)                                              │
│  Output: 50 messages to Kafka per cycle                                    │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ CallbackEventConsumer ─────────────────────────────────────────────────────┐
│                                                                             │
│  @Component with @KafkaListener                                            │
│  Listens to all 7 vendor queue topics                                      │
│  Receives messages in real-time                                            │
│  Looks up config by topic name                                             │
│  Calls repository to store data                                            │
│                                                                             │
│  When: Real-time (whenever message arrives)                                │
│  Output: Rows saved to database                                            │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ CallbackEventJdbcRepository ───────────────────────────────────────────────┐
│                                                                             │
│  @Repository for data persistence                                          │
│  Auto-creates target table if missing                                      │
│  Validates columns against schema                                          │
│  Builds parameterized INSERT (SQL injection safe)                          │
│  Executes database insert                                                  │
│                                                                             │
│  When: Called by consumer                                                  │
│  Output: Data persisted to _producer table                                 │
└─────────────────────────────────────────────────────────────────────────────┘
